<objective>
Define the local data backbone for the app by separating queryable records, repository interfaces, and lightweight settings storage before timer and task features start writing real data.

Purpose: Make local persistence durable and replaceable so future analytics, history, and timer recovery work do not couple directly to SwiftUI views or ad hoc storage choices.
Output: Domain models, SwiftData record models, a shared model-container factory, repository boundaries, a typed settings store, and persistence-focused tests.
</objective>
